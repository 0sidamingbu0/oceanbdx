# 2026-07-06 sim2sim 对齐路线 B：41 维 -> 57 维 path frame 观测

## 背景

`oceanisaaclab` 已按迪士尼 BDX 论文完整复刻路线 B（见 oceanisaaclab
`changelog/2026-07-06.md`），观测从旧 41 维（ang_vel3+proj_g3+cmd3+gait_clock2+
Δq10+dq10+last_act10）重构为 **57 维**，动作解码从全局 `action_scale` 改为逐关节
线性映射。训练侧导出的 ONNX 已是 `in=[1,57]`，旧 41 维布局的 sim2sim 直接加载会
因输入维度不匹配而无法运行。

本轮**只改 Python sim2sim（`sim2sim/mujoco_sim.py`）**，path frame 用 MuJoCo 真值
驱动；C++ 部署侧（`src/policy.cpp` / `include/oceanbdx/policy.hpp`）仍是旧 41 维，
列为待办（涉及状态估计器、改动大且需交叉编译验证，留待部署阶段）。

## 涉及文件

- `sim2sim/mujoco_sim.py`
- `config/oceanbdx.yaml`
- `policy/policy.onnx`（换为 57 维，旧 41 维备份 `policy/policy_41d_backup.onnx`）

## 57 维观测布局（须与训练 `_get_observations` 逐位一致）

```
[0:2]   pos_pf   × pos_pf_scale(4.0)      path 系躯干 xy
[2:4]   (sin, cos) 相对 yaw = head_yaw − path_yaw
[4:7]   lin_vel_b × lin_vel_scale(2.0)    body 系线速度
[7:10]  ang_vel_b × ang_vel_scale(0.25)   body 系角速度（= gyro）
[10:20] (q − default) × dof_pos_scale(1.0)
[20:30] dq × dof_vel_scale(0.05)
[30:40] a_{t-1}
[40:50] a_{t-2}                            ← 新增第二帧动作历史
[50:54] (sin2πφ, cos2πφ, sin4πφ, cos4πφ)   ← 相位二阶谐波
[54:57] cmd × commands_scale(2.0, 2.0, 1.0)
```

## 具体改动

### `sim2sim/mujoco_sim.py`

- 新增 `PathFrameNP`：path frame（论文 V-A / Fig.4）的单 env numpy 复刻，逐行为对齐
  训练侧 `path_frame.py`——行走按 path 系命令积分、站立一阶低通收敛到双脚中心+躯干
  朝向、最大偏差投影（位置 0.25m / yaw 0.6rad）拉回躯干附近。
- 新增辅助：`yaw_from_quat`（四元数取 yaw）、`wrap_angle`、`Sim.path_frame_truth()`
  （从 MuJoCo 读 path frame 所需真值：躯干 xy=`qpos[0:2]`、head_yaw=yaw(`qpos[3:7]`)+
  offset、双脚中心=两脚 geom `geom_xpos` 均值、body 线速度=`rot_wb.T@qvel[0:3]`）。
  **真机侧这些量须改由状态估计器/里程计 + FK 提供**（论文 V-D 要求 runtime 一致）。
- `head_yaw offset`：`forward_vx_sign=-1` → offset=π（URDF base +x 指尾部，头前向=−x）。
- `Policy` 类重写为 57 维：
  - `step` 签名扩展为 `(q, dq, gyro, cmd, pos_pf, yaw_pf, lin_vel_b)`；
  - 相位积分改用 φ̇ 恒定步频（`1/gait_cycle_period`），在 obs 构造**之前**推进
    （对齐训练 `_pre_physics_step` 先于 `_get_observations`）；
  - **站立时相位谐波不再清零**（删掉旧 41 维的 gait_clock 置零逻辑，训练侧路线 B
    不清零，否则观测分布与训练不一致）；
  - 相位谐波为二阶（sin/cos 2πφ + 4πφ）；新增 `last_last_actions` 第二帧动作历史，
    每步滚动 t-1 → t-2；
  - 动作解码从全局 `default + action_scale×a` 改为**逐关节** `default +
    action_joint_ranges ⊙ clip(a, ±1)`。
- `Sim.__init__` 创建 `PathFrameNP`、读 `forward_vx_sign`/`move_command_threshold`/
  path_frame 参数、缓存双脚 geom id。
- `control_step` RL 分支：进入 RL_BALANCE 时 `path_frame.reset` 到当前躯干位姿；
  每策略步先用 MuJoCo 真值 `path_frame.step`，再 `base_in_path_frame` 取 pos_pf/yaw_pf
  喂 `policy.step`。
- `probe_policy`：理想直立零命令下 pos_pf=0 / yaw_pf=0 / lin_vel_b=0。

### `config/oceanbdx.yaml`

policy 段：
- `num_obs` 41 → **57**；
- 新增：`pos_pf_scale: 4.0`、`lin_vel_scale: 2.0`、`forward_vx_sign: -1.0`、
  `path_frame_stand_time_constant: 1.0`、`path_frame_max_pos_deviation: 0.25`、
  `path_frame_max_yaw_deviation: 0.6`、
  `action_joint_ranges: [0.35,0.35,0.8,0.9,0.8, 0.35,0.35,0.8,0.9,0.8]`；
- `action_scale: 0.5` 保留字段但路线 B 已弃用（改逐关节映射），代码不再读。

## 验证

- `python3 sim2sim/mujoco_sim.py --probe-policy`：57 维 obs 拼装无维度错、ONNX 前向
  正常、输出 10 维动作合理未饱和。
- **path frame 对拍**：`PathFrameNP` vs 训练 torch `PathFrame` 同初始状态 + 同命令序列
  演化 50 步，max|Δpos|=6.7e-8、max|Δyaw|=1.8e-7、max|Δobs|=1.8e-7（浮点噪声）。
- **57 维 obs 对拍**：随机状态下 `Policy.step` 拼的 obs 与训练布局逐段核对，max|Δ|=0。
- **headless 闭环**：`--debug-push-steps 40`，起立 → RL_BALANCE，path frame + 57 维
  策略前向 40 步不崩，机器人站稳（z=0.407、倾角小、动作不饱和）。
  注：当前用的是训练 3000 iter 尚未学会迈步的策略（站立局部最优，见 oceanisaaclab
  changelog 第 13 节），故站着不动是预期，sim2sim 管线本身已验证正确；待重训出会迈步
  的策略后重新导出覆盖 `policy/policy.onnx` 即可。
