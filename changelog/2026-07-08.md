# 2026-07-08 sim2sim 同步:阶段1 脖子+头部命令(57→77维, 动作10→14)

训练侧(oceanisaaclab)阶段1 给 route B 行走策略加了脖子(4关节)+4-DOF 头部命令,
观测 57→77、动作 10→14(后 4=脖子位置目标)。本次把 MuJoCo sim2sim 同步到位。
详见 oceanisaaclab/changelog/2026-07-08.md。

## 观测布局(77 维,须与训练侧 _get_observations 逐位一致)
```
[0:2]   pos_pf × pos_pf_scale
[2:4]   (sin,cos) yaw_pf
[4:7]   lin_vel_b × lin_vel_scale
[7:10]  ang_vel_b × ang_vel_scale (=gyro)
[10:20] (q_leg − default) × dof_pos_scale
[20:24] q_neck × dof_pos_scale           脖子 4 关节角(无 default 偏移)
[24:34] dq_leg × dof_vel_scale
[34:38] dq_neck × dof_vel_scale
[38:52] a_{t-1} (14)
[52:66] a_{t-2} (14)
[66:70] phase 二阶谐波
[70:73] cmd × commands_scale
[73:77] head_cmd × head_command_scale     (Δh, pitch, yaw, roll)
```

## 改动
1. **config/oceanbdx.yaml**:`num_obs` 57→**77**、加 `num_act: 14`;新增 `neck_action_joint_ranges`
   [0.8,0.8,1.2,0.7]、`neck_default_dof_pos`、`head_command_scale` [20,1,1,1]、`neck_kp/kd` 50/2;
   command 段加头命令范围 max_head_dh/pitch/yaw/roll(0.02/0.5/1.0/0.6)。
2. **sim2sim/mujoco_sim.py**:
   - `Policy`:构造加 `num_neck`;obs 拼装插入 q_neck/dq_neck(位置在 leg 之后)+ 尾部 head_cmd;
     动作历史 10→14;`step()` 加 neck_q/neck_dq/head_cmd 入参,返回 **(leg_target, neck_target)**。
   - 加模块级 `ACTION_JOINTS = LEG_JOINTS + NECK_JOINTS`(14),调试打印按此索引避免越界。
   - `__init__`:加 `head_cmd`(4)、`neck_target`、`neck_default`、`neck_kp/kd`、`max_head` 缓冲;
     Policy 构造传 `num_neck=len(NECK_JOINTS)`。
   - `control_step`:RL 步给 policy 传 neck_q/dq + head_cmd,收 (rl_target, neck_target);
     **脖子驱动从锁零位改为位置伺服跟随 neck_target**(RL 状态)/锁默认位(其余),kp50/kd2。
   - `reset`:neck_target/head_cmd 归零。
   - `probe_policy`:适配 step 新签名。
   - 键盘:i/k=点头pitch± j/l=摇头yaw± u/o=歪头roll± n/m=头高± h=头命令清零。
3. **export_ckpt_onnx.py**:无需改(obs_dim 从 checkpoint 自动推定,77 维自动适配)。

## 验证

- **待训练侧出 77维/14动作 的新 onnx 后**在 mujoco 里实跑验证(前进步态 + 按 i/k/j/l 看头动)。

# 2026-07-08(追加)sim2sim 同步阶段2:站立(perpetual)独立模型 + 双模型运行时互切

训练侧(oceanisaaclab)按论文 divide-and-conquer 又训了一个**独立的站立策略**
(`Ocean-BDX-StandPaper-Direct-v0`,74 维观测/14 动作,无相位,命令换成躯干位姿 4-DOF)。
本次把 MuJoCo sim2sim 改成**同时加载行走 + 站立两个模型**,运行时按键互切:
**按 1 起立后进站立模型(RL_STAND),按 2 切行走模型(RL_WALK),按 1 再切回站立**。
详见 oceanisaaclab/changelog/2026-07-08.md 的"阶段2"。

## 两个模型的观测差异(站立 74 维 vs 行走 77 维)
公共块(66 维)完全一致:pos_pf2 + yaw_pf2 + lin_vel3 + ang_vel3 + q_leg10 + q_neck4
+ dq_leg10 + dq_neck4 + a_{t-1}14 + a_{t-2}14。仅尾部不同:
```
行走(77):… + phase二阶谐波4 + cmd3(vx,vy,wz)×commands_scale + head_cmd4×head_command_scale
站立(74):… +                 torso_cmd4(h,pitch,yaw,roll)×torso_command_scale + head_cmd4×head_command_scale
```
即站立**去掉相位谐波4**、把**速度命令 cmd3 换成躯干位姿命令 torso_cmd4**。动作解码(14 维:
腿 default+range·clip、脖子 neck_default+range·clip)两模型完全一致。

## 改动
1. **config/oceanbdx.yaml**:policy 段加 `stand_path: policy/stand/policy.onnx`(站立模型)、
   `torso_command_scale: [10,1,1,1]`(躯干命令缩放,须与 oceanisaaclab_stand_env_cfg 一致)。
2. **sim2sim/mujoco_sim.py**:
   - `Policy`:构造加 `stand` 开关 + `torso_command_scale`;`step()` 加 `torso_cmd` 入参,
     按 `self.stand` 决定观测尾部(站立=torso4+head4 无相位 / 行走=phase4+cmd3+head4)。
   - `Sim.__init__`:改为加载 `walk_policy` + `stand_policy` 两个独立 ONNX;`self.policy`
     变成"当前活动模型"引用(control_step 里按状态切换,供调试/日志/reset 复用)。
     加 `torso_cmd`(4)+ `max_torso` 缓冲。找不到站立 onnx 时 RL_STAND 回退到行走模型。
   - FSM:`RL_BALANCE` → **`RL_STAND`**(用站立模型)。起立完成自动进 RL_STAND;
     新增 `_switch_rl_state()` 在 RL_STAND↔RL_WALK 间切换(清目标模型动作历史/相位、
     锁当前腿角防 PD 跳变、命令清零、rl_tick 归零使下一步立即推理,path frame 保持连续)。
   - 键盘:`1`=SIT 起立 / RL 中切站立模型;`2`=切行走模型;`3`=切站立模型(兼容旧键位)。
     站立躯干命令微调(仅 RL_STAND 生效):`t/g`=pitch± `v/c`=yaw± `y/b`=roll± `f/z`=高度±。
   - `reset`/姿态保护/真机跟随增益/脖子伺服的状态判断:`RL_BALANCE`→`RL_STAND`。
   - CLI 加 `--stand-policy` 覆盖站立 onnx 路径。

## 验证
- mujoco_sim.py py_compile 通过;两个 onnx 维度实测:walk in[1,77]/out[1,14]、
  stand in[1,74]/out[1,14](input 名均为 `obs`),与代码布局一致。
- 无头冒烟测试:SIT→(1)→STAND_UP→自动 RL_STAND(站立模型推理 74 维)→(2)→RL_WALK
  (行走模型 77 维)→(1)→RL_STAND(74 维),**状态机切换与两模型推理维度全部正确**。
- 理想直立零命令探针:站立 |act|max=0.363、行走 0.224(同数量级、未饱和),观测布局无误。
- ⚠️ 当前拷入的站立 onnx 是 `model_300.pt` 导出(仅训到第 300 迭代,README 目标 20000),
  站立后会缓慢倾倒触发姿态保护 DAMPING——**属欠训练,非 sim2sim 代码问题**。
  训练侧多训几千迭代重导出到 `policy/stand/policy.onnx` 即可稳定。
