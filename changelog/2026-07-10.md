# 2026-07-10 行走观测升级、零速 phase 冻结与脖子步态对齐

## 行走观测 77 → 80：补充 projected gravity

### 背景

原行走策略的 77 维观测只通过 path-frame yaw 表达水平方向朝向，缺少能够直接
描述躯干 roll/pitch 倾斜的重力方向信息。训练侧新增了 body frame 下的三维
`projected_gravity`，因此行走策略观测由 77 维增加到 80 维。

站立策略没有增加该观测，接口仍为 74 维。当前两个策略接口分别为：

- walk：80 维观测、14 维动作。
- stand：74 维观测、14 维动作。

### 80 维行走观测布局

```text
[0:2]   pos_pf × pos_pf_scale
[2:4]   sin/cos(yaw_pf)
[4:7]   projected_gravity
[7:10]  lin_vel_b × lin_vel_scale
[10:13] ang_vel_b × ang_vel_scale
[13:23] q_leg - default
[23:27] q_neck
[27:37] dq_leg
[37:41] dq_neck
[41:55] action t-1
[55:69] action t-2
[69:73] phase 二阶谐波
[73:76] cmd
[76:80] head_cmd
```

### 改动

#### `sim2sim/mujoco_sim.py`

- `Policy.step()` 增加 `projected_gravity` 输入。
- 仅在 walk 观测的 `[4:7]` 插入三维重力投影，后续字段索引整体后移 3 位。
- MuJoCo 侧根据 base quaternion 计算 body frame 下的重力方向；机器人直立时约为
  `[0, 0, -1]`。
- walk ONNX 输入维度检查由 77 更新为 80，stand 仍检查 74 维。
- reset 和调试观测缓存同步保存 `projected_gravity`。

#### `config/oceanbdx.yaml`

- 行走策略 `num_obs` 更新为 `80`。
- 更新观测布局注释，标明新增的 `projected_gravity` 及其索引范围。

#### 模型与场景

- `policy/policy.onnx`、`policy/policy.onnx.data` 和 stand 模型文件随观测接口调整同步更新。
- `sim2sim/ocean_scene.xml` 同步了本次部署所需的场景配置。

### 兼容性

- 旧 77 维 walk checkpoint/ONNX 与新 sim2sim 观测接口不兼容，必须使用重新训练并
  导出的 80 维模型。
- stand 模型仍使用 74 维观测，不应向其中插入 `projected_gravity`。
- `src/policy.cpp` 当前仍是 legacy observation 部署链路，尚未完整支持新的
  80 维观测/14 维动作接口；现阶段应使用 Python sim2sim 验证行走模型。

## 背景

`oceanisaaclab` 训练侧发现：零速参考的腿、躯干、脖子和双脚接触虽然都是静态，
但步态 phase 仍在持续推进。行走策略因此在 `vx=vy=wz=0` 时仍收到周期变化的
四维相位谐波，sim2sim 表现为脖子和躯干持续周期扭动。

## 改动

### `sim2sim/mujoco_sim.py`

- 行走模型仅在 `max(|vx|, |vy|, |wz|) > move_command_threshold` 时推进
  `gait_phase`。
- 默认阈值为 `0.08`。命令低于或等于该阈值时冻结当前 phase，相位谐波
  保持常量，不再给零速策略持续的周期驱动。
- 非零速度仍按命令幅度在 `gait_cycle_period=0.6s` 与
  `gait_period_fast=0.48s` 之间插值，与训练侧的 `phase_rate(cmd)` 语义一致。

### `config/oceanbdx.yaml`

- 更新 `move_command_threshold` 注释：站立分支同时表示 path frame 收敛和步态
  phase 冻结。

## 与训练侧的同步关系

- `oceanisaaclab` 已将零命令 `phase_rate` 改为 `0`，并对 `vx/vy/wz` 统一使用
  `0.08` 命令死区。
- 行走时脖子和脚步共用同一 phase 及 `phase_rate(cmd)`。完整左右步态包含
  两次迈步，训练侧脖子参考改为二阶谐波，每只脚迈步时各起伏一次。
- 观测仍为 80 维，动作仍为 14 维，维度与排列未改变。

## 验证

- `python3 -m py_compile sim2sim/mujoco_sim.py` 通过。
- 观测构造单测：`gait_phase=0.37` 时连续两次零命令推理，phase 仍为
  `0.37`；切到 `vx=0.1` 后推进到 `0.4062`。
- 测试中行走观测仍为 80 维，ONNX 输入接口未发生维度回归。

## 兼容性

- phase 语义和训练侧脖子参考已改变，需要从头训练并重新导出行走 ONNX。
- 当前工作树中已有的 `policy/policy.onnx` 和 `policy/policy.onnx.data` 改动未被本次修改或回退。
- C++ `src/policy.cpp` 仍只支持旧观测接口，本轮只同步 Python sim2sim；真机部署前
  仍需单独完成 80 维/14 动作的 C++ 部署链路。

## 行走速度边界与默认头部运动修正

- sim2sim 原 `max_vx=0.5`、`max_vy=0.3`，分别是训练范围 `0.25/0.15` 的两倍。连续按键
  会把 80 维策略送入未训练命令区间，表现为步行几步后不断前倾并摔倒。
- `config/oceanbdx.yaml` 将行走命令上限收紧为 `vx=0.25 m/s`、`vy=0.15 m/s`，
  `wz=0.8 rad/s` 保持不变。
- 训练侧同步把满速前倾参考从 5° 降到 3°，并把默认周期脖子运动改为仅 `n1/n2`
  同向的小幅上下起伏；`n3/n4` 不再在零头命令时周期 yaw/roll。
- 观测/动作接口仍为 80/14，sim2sim 代码不需要额外动作解码改动；必须部署基于新参考
  重新训练和导出的 ONNX，旧模型不会因只修改 YAML 自动改变头部动作。
