# 2026-06-30 低速行走 v2 观测对齐：39 维 -> 41 维 gait clock

## 背景

`oceanisaaclab` 的低速行走 v2 已从旧站立/行走接口升级：

- 观测从 39 维变为 41 维。
- 在 `command` 后增加 `gait_clock = [sin(gait_phase), cos(gait_phase)]`。
- `commands_scale` 从 `[2.0, 2.0, 0.25]` 改为 `[2.0, 2.0, 1.0]`。
- 新策略训练包含 `vx` 与 `wz`，左右旋转命令应走 `command[2] = wz`。

因此 `/home/ocean/oceanbdx` 的 sim2sim 和 C++ 部署侧必须同步观测构造，否则 41 维 ONNX 会输入维度不匹配，或 yaw/gait 语义不一致。

- `sim2sim/mujoco_sim.py`

## 具体改动

### Python sim2sim

`sim2sim/mujoco_sim.py`：

- `Policy` 中新增：
  - `gait_cycle_period`
  - `policy_dt`
  - `gait_phase`
- `reset()` 时将 `gait_phase` 清零。
- `step()` 观测顺序改为：

```text
[
  gyro * ang_vel_scale,
  projected_gravity,
  command * commands_scale,
  sin(gait_phase), cos(gait_phase),
  (q - default_dof_pos) * dof_pos_scale,
  dq * dof_vel_scale,
  last_actions,
]
```

- 每次 policy step 后：

```text
gait_phase = (gait_phase + policy_dt / gait_cycle_period) % 1.0
```

- `policy_dt` 由 `control.dt * control.decimation` 自动传入。

### checkpoint 扫描工具

`sim2sim/scan_ckpt.py`：

- 与 `mujoco_sim.Policy` 同步 41 维观测。
- 从 `config/oceanbdx.yaml` 读取真实 `control.dt` 和 `control.decimation`，保证扫描 checkpoint 时 gait phase 推进速度与 sim2sim 一致。

### C++ 部署侧

`src/policy.cpp`：

- `PolicyRunner::Impl` 新增 `gait_phase`。
- `PolicyRunner::Reset()` 清零 `gait_phase`。
- `PolicyRunner::Step()` 观测从 `9 + 3*num_joints` 改为 `11 + 3*num_joints`。
- 在 command 后拼接 `sin(gait_phase)` / `cos(gait_phase)`。
- `gait_phase` 使用：

```text
policy_dt = control_dt * decimation
gait_phase = fmod(gait_phase + policy_dt / gait_cycle_period, 1.0)
```

### 配置

`config/oceanbdx.yaml`：

```yaml
policy:
  num_obs: 41
  commands_scale: [2.0, 2.0, 1.0]
  gait_cycle_period: 0.6
```

`include/oceanbdx/config.hpp` / `src/config.cpp`：

- `commands_scale` 默认改为 `[2.0, 2.0, 1.0]`。
- 新增 `gait_cycle_period` 配置读取。
